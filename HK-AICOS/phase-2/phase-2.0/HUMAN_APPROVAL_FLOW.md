# HUMAN APPROVAL FLOW
## HK-AICOS 人工確認流程

---

**版本：** v2.0.0
**日期：** 2026-05-13
**狀態：** Phase 2.0 文件骨架

---

## 核心原則

> **AI 只提供分析建議，人類 PM 作最終決定。**
> **AI 不可代表公司作任何正式承諾或決定。**

---

## 確認流程總覽

```
AI 生成分析報告
        ↓
風險等級評估
        ↓
┌───────────────────────────────────────┐
│ L1 低風險  → PM 審閱 → 直接輸出        │
│ L2 中風險  → PM 確認 → 輸出            │
│ L3 高風險  → PM + Legal 確認 → 輸出    │
│ L4 極高風險 → PM + Legal + 專業人士    │
│              確認 → 輸出               │
└───────────────────────────────────────┘
        ↓
人類 PM 簽發最終報告
```

---

## L1 低風險確認流程 🟢

### 適用情況

- 一般工程進度更新
- 日常材料採購
- 一般工地日報
- 輕微工序調整

### 確認步驟

```
步驟 1：PM 收到 AI 分析報告
步驟 2：PM 審閱報告（目標：< 24 小時）
步驟 3：PM 確認無問題
步驟 4：直接輸出 / 存檔
```

## L3 高風險確認流程 🔴

### 適用情況

- 工程延誤（> 7 天）
- 大額 VO 申請（HK$50,000 - HK$500,000）
- 輕微工傷
- 合約爭議（初步）
- 政府部門查詢

### 確認步驟

```
步驟 1：PM 收到 AI 分析報告
步驟 2：PM 審閱報告及風險提示（目標：< 2 小時）
步驟 3：PM 初步確認
步驟 4：Legal Agent 審查法律風險
步驟 5：PM + Legal 共同評估

步驟 7：PM 確認並簽署
步驟 8：輸出最終報告（附風險提示）
```

## L4 極高風險確認流程 ⚫

### 適用情況

- 工傷事故（需住院）
- 嚴重工傷 / 死亡事故
- 結構安全問題
- 電力事故
- 消防系統問題
- 法律訴訟
- 大額索償（> HK$500,000）

### 確認步驟

```
步驟 1：立即電話通報 PM（不可只發訊息）
步驟 2：PM 立即啟動緊急程序
步驟 3：AI 緊急分析（同步進行）
步驟 4：Safety Agent + Legal Agent 緊急分析
步驟 5：PM 立即確認（目標：< 30 分鐘）
步驟 6：Legal Agent 立即審查
步驟 7：識別需要確認的香港合資格專業人士
步驟 8：聯絡相關專業人士
步驟 9：等待專業人士確認
步驟 10：專業人士確認後才採取行動
步驟 11：記錄所有決策及確認
步驟 12：輸出最終報告
```

## 香港法定通報要求

### 工傷事故通報

| 事故類型 | 通報部門 | 通報時限 | 通報方式 |
|---------|---------|---------|---------|
| 工傷（需住院） | 勞工處 | 24 小時內 | 書面通報 |
| 死亡事故 | 勞工處 | 立即 | 電話 + 書面 |
| 危險事故 | 勞工處 | 24 小時內 | 書面通報 |

### 結構問題通報

| 問題類型 | 通報部門 | 通報時限 |
|---------|---------|---------|
| 結構危險 | 屋宇署 | 立即 |
| 斜坡問題 | 土木工程拓展署 | 立即 |

### 消防問題通報

| 問題類型 | 通報部門 | 通報時限 |
|---------|---------|---------|
| 消防系統失效 | 消防處 | 立即 |
| 危險品事故 | 消防處 | 立即 |

## 注意事項

⚠️ **AI 分析報告不是正式文件，不可直接對外發送**

⚠️ **所有正式文件必須由人類 PM 審閱及簽署**

⚠️ **緊急情況必須先通報，不可等待 AI 分析**

⚠️ **L4 極高風險事項必須獲得香港合資格專業人士確認後才採取行動**

⚠️ **所有確認記錄必須存檔，以備審計**
